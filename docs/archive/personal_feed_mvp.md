## 목적

`checkjjaek4`의 첫 구현 단위는 개인 feed 기반 글쓰기와 사용자 간 상호작용을 복원하는 것이다.
그룹은 중요한 기능이지만 첫 단계의 중심축이 아니라, 이후 붙는 승인형 확장 레이어로 둔다.

## 포함 범위

- Devise 기반 사용자 인증
- 개인 feed 글 작성/조회/수정/삭제
- 사용자 프로필 페이지
- 팔로우
- 좋아요
- 댓글
- Pundit 정책 기반 권한 확인

## 핵심 규칙

- 사용자 인증은 Devise gem을 이용해 처리한다.
- 개인 글 기본 visibility는 `public` 단일값으로 시작한다.
- 홈 feed는 `나 + 내가 팔로우한 사용자들의 글`만 기본 노출한다.
- 프로필 페이지에서는 해당 사용자의 공개 글을 조회할 수 있다.
- 댓글과 좋아요는 사용자가 접근 가능한 글이면 허용한다.
- 팔로우는 상호작용 선행조건이 아니라 feed 노출 규칙이다.
- 글 수정/삭제는 작성자만 가능하다.

## 제외 범위

- 그룹 개설 승인
- 그룹 멤버십
- 그룹 글을 개인 feed로 가져오기
- 책 검색과 책 연결
- 태그
- 메시지/사진 전용 모델

## 테스트 기준

- 비로그인 사용자는 인증이 필요한 개인 feed 흐름에 접근할 수 없다.
- 로그인 사용자는 자신의 글을 생성/수정/삭제할 수 있다.
- 홈 feed에는 본인과 followee 글만 보인다.
- 접근 가능한 글에는 댓글과 좋아요를 남길 수 있다.
- 자기 자신 팔로우와 중복 좋아요는 허용하지 않는다.
- Devise 로그인/로그아웃 이후 기본 진입 흐름이 깨지지 않아야 한다.

## 현재 반영 상태

- Devise 기반 로그인, 로그아웃, 회원가입 이후 기본 진입 흐름을 request spec으로 고정했다.
- 홈 feed는 나와 followee의 공개 글만 노출되도록 request/policy spec으로 고정했다.
- 프로필 페이지는 팔로우 여부와 무관하게 해당 사용자의 공개 글을 보여주도록 request/policy spec으로 고정했다.
- 글 수정/삭제와 댓글 수정/삭제는 작성자만 가능하도록 request/policy spec으로 고정했다.
- 자기 자신 팔로우 금지, 중복 좋아요 금지, 없는 like/follow 제거 시의 안전한 응답 흐름을 request spec으로 고정했다.
- 댓글 생성/수정 실패 시에는 같은 페이지를 `422`로 다시 렌더하도록 정리했다.
