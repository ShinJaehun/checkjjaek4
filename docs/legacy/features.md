# Legacy Features — checkjjaek3


## 상태

이 문서는 `checkjjaek3`의 사용자 흐름을 보관하는 참고 문서다.

현재 `checkjjaek4`의 활성 기능 기준은 아래 두 문서다.

- `docs/specs/bookjjaek_reboot_spec.md`
- `docs/reboot/reboot_plan.md`

이 문서의 내용은 현재 구현 방향보다 우선하지 않는다.

---

## 목적

이 문서는 `checkjjaek3`에서 사용자에게 실제로 제공하던 핵심 기능과 흐름을 정리하고,
`checkjjaek4`에서 무엇을 먼저 복원해야 하는지 판단하기 위한 기준 문서다.

원칙:

- 화면 수나 컨트롤러 액션 수보다 사용자 흐름을 우선한다.
- 레거시 구현 세부를 그대로 재현하는 것이 목적이 아니다.
- `checkjjaek4`에서는 핵심 사용자 가치가 유지되는 범위에서 더 단순한 구조를 우선한다.
- 기능 우선순위를 정하는 문서이지, 구현 방법을 확정하는 문서는 아니다.

---

## 레거시 핵심 사용자 흐름 요약

`checkjjaek3`의 기능은 크게 아래 흐름으로 나눌 수 있다.

1. 사용자 가입/로그인 후 프로필을 가진다
2. 그룹을 만들거나 그룹에 가입한다
3. 책을 검색하고 책과 관련된 글을 남긴다
4. 글을 올리고 댓글/좋아요 등으로 상호작용한다
5. 사용자 간 팔로우를 통해 피드 맥락을 형성한다
6. 태그와 그룹, 책, 사용자 맥락을 따라 콘텐츠를 본다

---

## 기능별 정리

## 1. 인증 / 사용자 계정

### 레거시 기능
- 회원가입
- 로그인 / 로그아웃
- Devise 기반 계정 관리
- 사용자 프로필 조회
- 프로필 이미지 표시

### 사용자 가치
- 서비스의 모든 활동은 사용자 계정을 기준으로 연결된다.
- 포스트, 댓글, 좋아요, 그룹 가입, 팔로우의 출발점이다.

### checkjjaek4 판단
가장 먼저 복원해야 하는 기능이다.

초기에는:
- 로그인/로그아웃
- 최소 프로필
- 사용자 식별

까지만 먼저 가져오고,
세부 프로필 편집은 뒤로 미뤄도 된다.

---

## 2. 그룹 생성 / 가입 / 승인

### 레거시 기능
- 그룹 생성
- 그룹 목록/상세
- 그룹 가입 요청
- 가입 승인 또는 상태 전환
- 그룹 소속 사용자 목록

### 관련 모델
- Group
- UserGroup
- group_state
- user_groups.state

### 사용자 가치
- 책짹은 단순 개인 기록 앱이 아니라,
  그룹 기반 독서/소통 공간 성격이 강하다.
- “어떤 그룹에 속해 있는가”가 사용자 경험의 중요한 축이다.

### checkjjaek4 판단
핵심 기능이다.

오히려 레거시 분석상,
이 기능은 Post보다도 먼저 구조를 잡아야 할 가능성이 높다.

초기 복원 우선순위:
- 그룹 생성
- 그룹 보기
- 그룹 가입 요청
- 멤버십 상태 반영

---

## 3. 책 검색 / 책 연결

### 레거시 기능
- 외부 책 검색 API 연동
- 검색 결과를 Book 모델로 연결
- 책과 연결된 글 작성 흐름

### 사용자 가치
- “책 기반 커뮤니티”라는 서비스 정체성과 직접 연결된다.
- 단순 자유 게시판과 구분되는 요소다.

### checkjjaek4 판단
핵심 기능이지만, 인증/그룹/포스트 뼈대 다음 단계가 적절하다.

레거시 `checkjjaek3`는 외부 책 검색 API를 사용해 책 정보를 조회하고,
검색 결과를 서비스 내 Book 흐름과 연결했다.

이 기능은 서비스 정체성과 직접 연결되므로 `checkjjaek4`에서도 유지 가치가 높다.
다만 구현은 레거시 코드를 그대로 재사용하지 않고,
현재 시점의 공식 API 문서를 다시 확인한 뒤 재설계한다.

초기 방향:
- 먼저 책 검색 결과를 보여주고
- 필요한 책 정보만 저장하거나 참조하고
- Post와 연결하는 흐름을 만든다
- 레거시처럼 책이 `postable`이어야 할 필요는 없다.

---

## 4. 포스트 작성 / 조회

### 레거시 기능
- 글 작성
- 글 수정/삭제
- 그룹/사용자/책/메시지/사진 맥락의 포스트
- 해시태그 추출
- 피드형 노출

### 사용자 가치
- 서비스의 실질적인 중심 기능이다.
- 독서 기록, 감상, 추천, 대화가 모두 포스트를 통해 표현된다.

### checkjjaek4 판단
핵심 기능이다.

다만 구조는 단순화한다.

초기 방향:
- `Post` 단일 모델 중심
- 책 연결은 optional
- 사진은 첨부
- 메시지성/수신자성 포스트는 별도 모델보다 Post 속성으로 먼저 검토

즉, 기능은 유지하되 레거시 `postable` 설계는 그대로 가져오지 않는다.

---

## 5. 댓글 / 대댓글

### 레거시 기능
- 포스트에 댓글 작성
- 댓글에 답글 작성
- parent_id 기반 계층 구조

### 사용자 가치
- 단순 피드가 아니라 상호작용 구조를 만들어 준다.

### checkjjaek4 판단
핵심 기능이다.

다만 초기에는:
- Post 전용 댓글
- 대댓글 1단계 정도

로 단순화해도 충분할 가능성이 높다.

---

## 6. 좋아요

### 레거시 기능
- Post에 좋아요
- 사용자별 좋아요 기록

### 사용자 가치
- 가벼운 반응/관심 표현
- 피드 참여감 상승

### checkjjaek4 판단
중요하지만 초기 필수는 아니다.

Post/Comment/Group 흐름이 먼저 자리잡아야 한다.
좋아요는 그 다음에 붙이기 쉬운 기능이다.

---

## 7. 팔로우

### 레거시 기능
- 사용자 간 팔로우
- follower / followee 관계
- 팔로우 기반 피드 맥락

### 사용자 가치
- “내가 관심 있는 사람의 글”이라는 흐름을 만든다.

### checkjjaek4 판단
서비스 정체성에는 의미가 있지만,
초기 재구축의 최우선 기능은 아니다.

초기에는 그룹 기반 흐름을 먼저 세우고,
팔로우는 후순위로 미뤄도 된다.

---

## 8. 태그 / 해시태그

### 레거시 기능
- 포스트 본문에서 해시태그 추출
- Tag 모델과 연결
- 태그별 조회

### 사용자 가치
- 콘텐츠 탐색성 향상
- 주제 기반 분류

### checkjjaek4 판단
후순위 기능이다.

초기에는 없어도 서비스 핵심 흐름은 성립한다.

---

## 9. 사진 포스트

### 레거시 기능
- Photo 모델
- 여러 이미지 첨부
- Photo에 Post가 붙는 구조

### 사용자 가치
- 이미지 중심 게시물 작성

### checkjjaek4 판단
기능은 유지 가치가 있다.

하지만 구현은 단순화한다.

초기 방향:
- Photo 별도 모델 대신 Post에 이미지 첨부

---

## 10. 메시지 포스트

### 레거시 기능
- Message 모델이 존재
- sender/receiver 개념이 있었던 흔적 존재
- 실제로는 Message에 Post가 붙는 구조

### 사용자 가치
- 특정 대상에게 전달하는 글/메시지 흐름을 의도한 것으로 보임

### checkjjaek4 판단
초기 핵심 기능으로 보기 어렵다.

메시지가 정말 독립 도메인인지,
아니면 “대상이 있는 Post”로 표현 가능한지를 먼저 검토한다.

현재 판단으로는 후자 가능성이 더 높다.

---

## 11. 사용자/그룹/책 맥락의 피드

### 레거시 기능
- 사용자별 포스트
- 그룹 관련 포스트
- 책 관련 포스트
- 태그/팔로우/좋아요와 결합된 탐색 흐름

### 사용자 가치
- 앱이 단순 CRUD가 아니라 커뮤니티처럼 느껴지게 하는 핵심 맥락

### checkjjaek4 판단
장기적으로 중요하다.

다만 초기에는 아래처럼 순서를 나눠야 한다.

1. Post 작성/조회
2. Group 맥락의 Post
3. Book 연결 Post
4. 그 다음 사용자/팔로우/태그 기반 피드 확장

## checkjjaek4에서 바로 복제하지 않을 것

아래는 레거시 기능이더라도 초기에 그대로 재현하지 않는다.

- `postable` polymorphic에 의존한 포스트 분기
- CanCanCan ability 기반 권한 흐름
- Rolify role/resource 구조
- Webpacker/Turbolinks 전제 UI
- 태그 자동 추출까지 포함한 복잡한 포스트 저장 흐름
- 메시지 전용 모델 구조
- 사진 전용 모델 구조

---

## 현재 기준 핵심 사용자 가치

`checkjjaek4`에서 먼저 복원해야 할 사용자 가치는 아래로 요약할 수 있다.

1. 로그인한 사용자가
2. 그룹이라는 맥락 안에서
3. 책과 연결된 글을 남기고
4. 다른 사람과 댓글로 상호작용할 수 있어야 한다

이 핵심이 성립한 뒤에
좋아요, 팔로우, 태그, 고급 피드를 붙이는 순서가 적절하다.
